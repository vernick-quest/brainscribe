# BrainScribe — Overnight Audit (2026-06-21)

Three read-only deep dives: Security & access-control, COPPA flow, Accessibility.
Nothing was changed by the audits. Severities: Critical / High / Medium / Low.

## 🚨 Must-fix before beta opens to real parents
1. **COPPA: the promised 7-day auto-deletion does not exist** (COPPA-C1). The consent email, Privacy Policy, and Terms all promise an under-13 account is auto-deleted if not approved in 7 days. There is **no cron / scheduled job / edge function** that does this — verified across migrations, `vercel.json` (no `crons`), no `supabase/functions/`, no `pg_cron`. Expiry only flips a status string *if a parent re-opens the dead link*; the `profiles` row + the child's name/email/avatar persist forever. → Build a Vercel Cron (or pg_cron) that deletes `pending_coppa_signups` where `status='pending' AND expires_at < now()` via `auth.admin.deleteUser` (cascades). Until then the "has been automatically deleted" copy is false.
2. **COPPA consent isn't bound to the invited parent** (SEC-C1 == COPPA-#4). `app/coppa/complete/page.js` selects `parent_email` but never checks it against the approver, and only blocks signers whose role is already `student`. Anyone with the token + any fresh Google account (a child can make one) can self-approve, and the audit log records the wrong person. → Require `user.email === pending.parent_email` (case-insensitive) AND block `user.id === pending.student_id` before approving.

## 🔐 Security & access-control

**CRITICAL**
- **C1 — COPPA consent binding flaw.** `app/coppa/complete/page.js:24-105`. (See item 2 above.) Fix: match signer email to `pending.parent_email`; make the UPDATE status-guarded/idempotent.

**MEDIUM**
- **M1 — Open redirect in OAuth callback.** `app/api/auth/callback/route.js:8,19` — `next` is used as `${origin}${next}` without validating it's a local path; `next=//evil.com` → off-site redirect (phishing). Fix: reject `next` unless it starts with a single `/` (not `//` or `/\`). Apply to the `/welcome?next=` reuse too.

**LOW**
- **L1 — Transcript role spoofing.** `app/api/messages/route.js:8-12` — client can POST `role:'assistant'|'system'` into their own session (RLS scopes to owner; no cross-user impact, but fabricates the transcript the LLM/teacher reads). Fix: force `role:'user'` on this endpoint.
- **L2 — Self-declared teacher/parent role.** `app/api/profile/confirm-role` lets a 13+ user set their own role. Not exploitable today (teacher data needs an `assignment_teachers` link), but document the invariant. 
- **L3 — Denial-of-wallet.** `app/api/tutor/route.js` and `…/summary` call the model on client-supplied text without an ownership short-circuit (rate-limited, so cost-only). Optional: 403 when the session isn't readable.

**Schema drift (commit as migrations):** `is_admin()` (used by RLS in 006/004) and `invites.assignment_id`/`expires_at` are referenced but not in any migration — they live only in the live DB. Fail-closed, but not reproducible.

**Solid:** all `/api/admin/*` verify admin server-side before privileged ops; `delete-user` blocks self-delete; impersonation only honored for DB-verified admins (forged cookie → null); `/profile/[studentId]` checks membership before fetch (no IDOR); invites verify session ownership; service-role key only server-side (no client leak); RLS policies sound; rate limiting on all expensive endpoints.

## 🧒 COPPA / under-13 flow

- **CRITICAL #1 — 7-day auto-deletion missing.** (Item 1 above.) Promises in `coppa/initiate:121`, `resend:86`, `welcome:376`, `privacy`, `terms`; no mechanism anywhere. Build the cron.
- **HIGH #2 — Expiry is a lazy side-effect.** `coppa/consent/page.js:37-50` only sets `status='expired'` when someone visits the link; never deletes. Same cron fixes it.
- **MEDIUM-HIGH #3 — Under-13 can re-declare 13+ to bypass.** `confirm-role` overwrites `age_bracket` to `13plus` without clearing/re-checking `coppa_consent_required`; the sessions/`assignment/new` gates check `age_bracket` not `coppa_consent_required`, so flipping the bracket lets them create coach sessions (dashboard still blocks, so they're "half-in"). Fix: refuse under13→13plus upgrade; make `coppa_consent_required && !given` the authoritative gate everywhere.
- **MEDIUM #4 — No proof the signer is the parent.** (== SEC-C1.) Block self-consent + match `parent_email`.
- **LOW #5-7 — Honesty/validation:** "has been deleted" copy is false until the cron exists (reword to "will be"); `parent_email` only checked for `@`; student-email display falls back to parent email cosmetically.

**Solid:** under-13 forced to role=student server-side; consistent blocking at dashboard/onboarding/assignment-new/sessions for the normal path; `/coppa/consent` correctly public; strong unguessable tokens; idempotent re-click handling; thorough approval transaction (parent role + watcher link + audit log + student email); rate-limited initiate/resend.

## ♿ Accessibility (new UI)

**Two global CSS fixes resolve most of it:**
- **G1 (CRITICAL) — No visible keyboard focus anywhere.** No `:focus-visible` rule exists; `--focus-ring` is defined (globals.css:122) but never applied; inputs use `outline:none`. Keyboard/ADHD students can't see where they are. Fix: global `:focus-visible { outline: 2px solid var(--ring); outline-offset: 2px }` + drop bare `outline:none`.
- **G2 (HIGH) — No `prefers-reduced-motion`.** Looping `animate-ping`/`animate-pulse` (mic, typing dots, caption), onboarding dot animation, spinners all ignore the OS setting. Fix: add a reduced-motion media block.

**Contrast (verified ratios, AA needs 4.5:1 text / 3:1 large+UI):**
- `--text-subtle` `#B2A994` → **2.25:1 FAIL all sizes** (used for real text: row meta, timestamps, "Your writing coach", transcript notes). Darken to ~`#7A7160`.
- `--text-muted` `#8C8474` → **3.58:1**, fails small text (filter pills, "Teacher" label, most secondary text, transcript section h2s). Darken to ~`#6F6857`.
- `--accent` as text/icon `#F0811E` → **2.67:1 FAIL** (replay/edit/assemble labels, status badges, icons). Use `--orange-600`-ish for text/icons; keep bright fill for backgrounds.
- status-success / status-thin small text, and white-on-`#D1503C` delete label (4.27:1) fail small. (Banner white-on-`#C0271E` = 5.92:1 PASS; navy bubbles 12:1 PASS.)

**Component criticals/highs:**
- **N1 (CRITICAL) — Upload box is a clickable `<div>`** (`NewSessionForm.js:110-156`): no role/tabIndex/key handler → keyboard + SR users **cannot upload a worksheet**. Make it a `<button>` (or role+tabIndex+onKeyDown) with an accessible name; keep drag-drop as enhancement.
- **Tap targets < 44px** (design mandates 44): ⋯ overflow 32px, delete trash 28px, expand chevron ~20px, filter pills ~31px, role select ~20px, replay ~18px, coach-switcher 24px, subject/teacher chips ~30px, many TutorSession action buttons. Bump hit areas.
- **Popovers not keyboard-operable** (SessionsList ⋯/picker, TutorSession coach-switch, subject/teacher): close only via a `<div>`/`mousedown` catcher — no Esc, no focus move, no `aria-expanded`/`role=menu`.
- **Icon-only buttons missing `aria-label`** (title-only or none): mobile avatar-only teacher chip, rename "Save", remove "×", trash, coach-switch, mobile subject/teacher chips.
- **Hover-only edit buttons** (`TutorSession.js:1896,1976`): `opacity-0 group-hover` → invisible/unreachable on touch + keyboard. Reveal on `focus-within` + on coarse pointers.
- **Onboarding dots** not announced to SR (add `role="progressbar"`/sr-only step text); selection states color-only (add `aria-pressed`).
- **`aria-live`** missing on streamed coach replies, mic state, upload status.

**Solid:** coach grid uses real buttons + `aria-pressed`; mic button well-sized + labeled + state-driven; LegalDoc fully semantic; chat bubbles high-contrast; chevrons/most icon buttons have `aria-label`; decorative SVGs `aria-hidden`.
